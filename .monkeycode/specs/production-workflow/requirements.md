# 生产工作流需求

## 引言

本需求将短剧脚本工坊从单次生成原型升级为可持续创作、可恢复任务、可追踪资产和可交付归档的单用户生产工具。

## 术语表

- **实体 ID**: 角色、场景、分镜和漫画格的不可变标识。
- **项目 Revision**: 每次成功持久化后递增的项目版本号。
- **来源 Revision**: 分析结果或媒体资产生成时对应的上游内容版本。
- **生成任务**: 图片、视频或分析工作的可持久化执行记录。
- **资产记录**: 包含生成参数、来源实体、版本和媒体地址的统一记录。
- **失效状态**: 上游内容变化后，下游结果需要确认或重新生成的状态。

## 需求

### 需求 1：稳定数据身份

**用户故事:** AS 创作者, I want 生成结果与业务实体稳定关联, so that 重命名、排序和增量编辑不会导致素材错绑

#### 验收标准

1. WHEN 系统加载历史项目, 系统 SHALL 为缺少 ID 的角色、场景、分镜和漫画格分配不可变 ID。
2. WHEN 用户重命名或移动实体, 系统 SHALL 保持实体 ID 与已有资产关联不变。
3. WHEN 异步任务完成, 系统 SHALL 通过项目 ID 和实体 ID 写入目标实体。

### 需求 2：可靠持久化

**用户故事:** AS 创作者, I want 连续编辑可靠保存, so that 刷新、切换项目和并发更新不会丢失内容

#### 验收标准

1. WHEN 用户连续更新多个字段, 系统 SHALL 合并待保存状态并完整持久化。
2. WHEN 项目版本与服务端版本冲突, 系统 SHALL 返回冲突响应并保留客户端待保存数据。
3. WHEN 系统写入项目文件, 系统 SHALL 使用原子替换保留完整 JSON。
4. WHILE 项目存在待保存操作, 系统 SHALL 显示保存状态。

### 需求 3：可恢复生成任务

**用户故事:** AS 创作者, I want 刷新后继续查看生成任务, so that 长时间视频任务不会失联

#### 验收标准

1. WHEN 图片或视频任务创建, 系统 SHALL 立即保存任务状态、目标项目、目标实体和生成参数。
2. WHEN 项目重新加载, 系统 SHALL 恢复未完成视频任务的轮询。
3. WHEN 任务完成, 系统 SHALL 同时更新任务、统一资产库和目标实体引用。
4. IF 任务失败或超时, 系统 SHALL 保存错误信息并提供重试所需参数。

### 需求 4：资产谱系与失效

**用户故事:** AS 创作者, I want 看清素材对应的设定版本, so that 我能判断素材是否需要重新生成

#### 验收标准

1. WHEN 上游文本、设定、Prompt、风格或画幅变化, 系统 SHALL 增加来源 Revision 并标记相关下游结果失效。
2. WHEN 媒体生成完成, 系统 SHALL 保存模型、参数、来源实体、来源 Revision、父资产和创建时间。
3. WHEN 用户重新生成媒体, 系统 SHALL 保留历史资产记录并将新资产设为当前引用。

### 需求 5：长文本与局部重跑

**用户故事:** AS 长篇创作者, I want 按章节分批生成并局部续跑, so that 单次上下文限制和局部失败不会阻断全部工作

#### 验收标准

1. WHEN 分析内容超过配置阈值, 系统 SHALL 按章节或文本段落拆分请求。
2. WHEN 分批结构化结果完成, 系统 SHALL 按稳定 ID 合并并重新编号镜头。
3. IF 某个分块失败, 系统 SHALL 保存成功分块与失败位置。
4. WHEN 用户执行单章分析, 系统 SHALL 保存章节结果并独立维护项目聚合结果。

### 需求 6：镜头连续性

**用户故事:** AS 导演, I want 相邻镜头传递视觉状态, so that 角色、道具和空间关系保持连续

#### 验收标准

1. WHEN 系统生成关键帧, 系统 SHALL 注入前一镜头连续性状态和可用参考帧。
2. WHEN 系统执行一致性检查, 系统 SHALL 返回包含严重级别、实体 ID、镜头 ID、规则和处理状态的问题列表。
3. WHILE 存在阻断级连续性问题, 系统 SHALL 在交付区域显示阻断状态。

### 需求 7：完整交付

**用户故事:** AS 制作人员, I want 导出可验证的生产清单, so that 剪辑和归档能够复现项目状态

#### 验收标准

1. WHEN 用户导出字幕, 系统 SHALL 按集生成独立 SRT 时间轴。
2. WHEN 用户导出生产清单, 系统 SHALL 包含项目 Revision、实体、资产、任务、Prompt 和校验信息。
3. WHEN 用户导出交付包, 系统 SHALL 提供包含 manifest、JSON、Markdown 和逐集字幕的归档文件。

### 需求 8：安全与运行边界

**用户故事:** AS 部署者, I want 明确单用户运行边界和受控 Provider 配置, so that 项目数据和调用额度得到保护

#### 验收标准

1. WHEN 服务接受 Provider URL, 系统 SHALL 校验 HTTPS 协议和允许的主机范围。
2. WHEN 服务处理生成请求, 系统 SHALL 应用请求大小、并发和输入长度限制。
3. WHERE 服务用于共享网络, 系统 SHALL 要求访问令牌保护 API。
